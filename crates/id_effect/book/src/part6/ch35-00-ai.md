# AI and MCP

Phase H adds `id_effect_ai` — vendor-neutral LLM traits over Phase A `HttpClient`.

## Language model trait

```rust
use id_effect_ai::{ChatRequest, LanguageModel, complete};
```

`LanguageModel` is a capability (`LanguageModelKey`). Install `MockLanguageModel` in tests; swap OpenAI-compatible HTTP in production.

## Streaming

`complete_stream` returns `Stream<CompletionChunk, AiError, R>` — token deltas map to `Chunk<String>` for sinks and UI bridges.

## MCP server template

`cargo run -p id_effect_ai --example mcp_server_template` scaffolds a minimal MCP JSON-RPC loop over stdio using the same `Effect` + capability patterns.

Secrets belong in `id_effect_config` — never log API keys.
